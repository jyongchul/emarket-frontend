## 🔴 Task #1: Cloudflare Hostname Routes 설정 (5분)

**우선순위**: 🔴 **CRITICAL** - 최우선

### 문제 상황

**증상**:
```
브라우저에서 https://wp-emarket.whmarketing.org 접속 시
→ HTTP 530 에러 발생
→ Cloudflare Error 1033
```

**실제 상황**:
- ✅ WordPress는 정상 작동 중 (Frontend에서 데이터 수신)
- ✅ Cloudflare Tunnel도 HEALTHY 상태
- ❌ 하지만 Hostname Routes 미설정으로 직접 접속 불가

**비유**: 건물은 정상 운영 중이지만, 정문 표지판이 없어서 직접 방문자는 들어갈 수 없음

---

### 해결 방법 (5분 소요)

#### Step 1: Cloudflare Dashboard 접속 (1분)

1. 브라우저에서 https://one.dash.cloudflare.com/ 접속
2. Cloudflare 계정으로 로그인
3. 좌측 메뉴에서 **"Networks"** 클릭
4. **"Routes"** 탭 선택
5. **"Hostname routes"** 탭 선택

**현재 URL**: https://one.dash.cloudflare.com/3eb25e0c75121d166641ecad0aa47f40/networks/routes

---

#### Step 2: Hostname Route 추가 (3분)

1. **"+ Add a hostname route"** 버튼 클릭

2. 다음 정보 입력:

```
┌─────────────────────────────────────────┐
│ Hostname                                │
│ wp-emarket.whmarketing.org              │
└─────────────────────────────────────────┘

┌─────────────────────────────────────────┐
│ Tunnel                                  │
│ [드롭다운에서 "emarket" 선택]             │
└─────────────────────────────────────────┘

┌─────────────────────────────────────────┐
│ Service                                 │
│ HTTP                                    │
└─────────────────────────────────────────┘

┌─────────────────────────────────────────┐
│ URL                                     │
│ localhost:80                            │
│ (또는 localhost:8080, localhost:8005)   │
│ (WordPress가 실행되는 포트 번호)          │
└─────────────────────────────────────────┘
```

3. **"Save hostname route"** 버튼 클릭

**스크린샷 필수**:
- 설정 입력 화면
- 저장 완료 화면

---

#### Step 3: 검증 (1분)

**1. 브라우저 테스트** (30초 대기 후):

